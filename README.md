# SauceDemo Test Automation Suite

## Overview

This project contains an automated regression test suite for the SauceDemo web application:

https://www.saucedemo.com/

The framework is implemented using Python, Playwright and Pytest following the Page Object Model (POM) design pattern.

The goal of this project is to verify the core functionalities of the application through automated end-to-end tests.

Github Actions are added.

---

## Technologies

* Python 3.10.5
* Playwright
* Pytest
* Pytest-HTML
* Git

---

## Framework Features

* Page Object Model (POM)
* Automated UI testing using Playwright
* Data-driven login tests using Pytest parameterization
* Automated HTML reporting using pytest-html
* Regression coverage for authentication, inventory, cart, checkout, navigation and logout workflows
* Git-based version control and reproducible test execution

## Installation

Clone the repository:

```bash
git clone https://github.com/salaarhamza/saucedemo-test-automation.git
cd saucedemo-test-automation
```

Create a virtual environment:

```bash
python -m venv venv
```

Activate the virtual environment:

### Windows

```bash
venv\Scripts\activate
```

Install dependencies:

```bash
pip install -r requirements.txt
```

Install Playwright browsers:

```bash
playwright install
```

---

## Running Tests

Run all tests:

```bash
pytest
```

The test suite automatically generates an HTML report after execution.

---

## Reporting Strategy

The framework uses the `pytest-html` plugin to automatically generate an HTML execution report.

After running:

```bash
pytest
```

the report is available at:

```text
reports/report.html
```

A sample HTML report is included in this repository to demonstrate the reporting output generated by the framework.

The console additionally provides the standard Pytest execution summary including passed, failed and skipped tests.

---

## Design Approach

The framework follows the Page Object Model (POM) design pattern.

Page interactions are separated from test logic to improve:

* Readability
* Maintainability
* Reusability

The `standard_user` is used for the primary end-to-end regression workflow because it represents the expected shop behavior. Additional SauceDemo users are covered through dedicated login validation tests.

---

## Assumptions

* Tests are executed against the public SauceDemo environment.
* Product data and product count are based on the current state of the application.
* Internet connectivity is required to access the test environment.
